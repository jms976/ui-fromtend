# UI Components Overview

UI 컴포넌트는 기능과 역할에 따라 체계적으로 분류되어 있습니다.<br/>
이 문서는 각 카테고리의 역할과 기준을 설명하여, 필요한 컴포넌트를 더 쉽고 빠르게 찾을 수 있도록 돕는 가이드입니다.
<br />
<br />

---
<br />

## MS Design System Storybook 설계 원칙

JUI 의 모든 컴포넌트는 다음 세 가지 핵심 원칙에 따라 설계되고 분류됩니다.

<ul>
  <li><strong>일관성 (Consistency)</strong> : 어떤 제품에서든 일관된 사용자 경험을 제공하기 위해, 컴포넌트의 시각적 스타일과 동작 방식을 통일합니다.</li>
  <li><strong>재사용성 (Reusability)</strong> : 다양한 상황에서 유연하게 재사용할 수 있도록 범용적인 컴포넌트를 설계하고, 불필요한 중복을 최소화합니다.</li>
  <li><strong>직관성 (Intuitiveness)</strong> : 컴포넌트의 이름과 구조만으로도 그 기능과 역할을 쉽게 예측할 수 있도록 직관적인 분류 체계를 지향합니다.</li>
</ul>

## packages/ui의 컴포넌트들 정리 기준

위 설계 원칙에 따라, 컴포넌트는 기능과 역할에 맞춰 대분류와 하위 소분류로 구성됩니다.<br/>
하단의 역할의 대분류에 대한 설명 및 구조입니다.
